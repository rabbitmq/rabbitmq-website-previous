Community Plugins

For your convenience, we offer binary downloads of plugins which have been contributed by authors in the community. These plugins do not ship with the RabbitMQ server itself; you can download them from our community plugin archive.

The plugins are available compiled against various releases of RabbitMQ.

Of course, we offer no guarantee that these will even work against a given release of RabbitMQ. We build them at release time, and also when requested to by a plugin author.

The download links on this page let you download the latest build of the plugin, compiled against the latest feature release of RabbitMQ. Bugfix releases do not affect plugin compatibility. For older versions of plugins or RabbitMQ, see the archive. See also how to install additional plugins

If you are the author of a plugin and would like it listed here, please get in touch!

Routing

rabbitmq_lvc
The last value exchange acts like a direct exchange (binding keys are compared for equality with routing keys); but it also keeps track of the last value that was published with each routing key, and when a queue is bound, it automatically enqueues the last value for the binding key.
  • Download: rabbitmq_lvc has not yet been built for unknown.
  • Maintainer: Simon MacMullen
  • Github: simonmacmullen/rabbitmq-lvc-plugin
rabbitmq_rtopic_exchange
Adds a reverse topic exchange which lets you provide routing patterns at publishing time, instead of at binding time.
  • Download: rabbitmq_rtopic_exchange has not yet been built for unknown.
  • Author: Alvaro Videla
  • Github: videlalvaro/rabbitmq-rtopic-exchange
rabbitmq_recent_history_exchange
A plugin that caches the last N messages that passed through an exchange and delivers them to newly bound queues.
  • Download: rabbitmq_recent_history_exchange has not yet been built for unknown.
  • Author: Alvaro Videla
  • Github: videlalvaro/rabbitmq-recent-history-exchange

Authentication / Authorisation

rabbitmq_auth_backend_http
Provides the ability for your RabbitMQ server to perform authentication (determining who can log in) and authorisation (determining what permissions they have) by making requests to an HTTP server.
  • Download: rabbitmq_auth_backend_http has not yet been built for unknown.
  • Author: Simon MacMullen
  • Github: simonmacmullen/rabbitmq-auth-backend-http
rabbitmq_auth_backend_amqp
Provides the ability for your RabbitMQ server to perform authentication and authorisation over AMQP itself.
  • Download: rabbitmq_auth_backend_amqp has not yet been built for unknown.
  • Author: Simon MacMullen
  • Github: simonmacmullen/rabbitmq-auth-backend-amqp

Management

rabbitmq_top
Adds a "top" like view of busy processes to the management plugin.
  • Download: rabbitmq_top has not yet been built for unknown.
  • Author: Simon MacMullen
  • Github: simonmacmullen/rabbitmq-top
rabbitmq_management_exchange
Adds an exchange type which allows you to connect to the management API using AMQP rather than HTTP.
  • Download: rabbitmq_management_exchange has not yet been built for unknown.
  • Author: Simon MacMullen
  • Github: simonmacmullen/rabbitmq-management-exchange
rabbitmq_event_exchange
Adds an exchange amq.rabbitmq.event which allows you to hear when queues, exchanges, bindings, users, connections and so on are created and deleted.
  • Download: rabbitmq_event_exchange has not yet been built for unknown.
  • Author: Simon MacMullen
  • Github: simonmacmullen/rabbitmq-event-exchange

Protocols

rabbitmq_email
Maps SMTP to AMQP (to convert an incoming email to an AMQP message) and AMQP to SMTP (to send an email from an AMQP message).
  • Download: gen_smtp has not yet been built for unknown.
  • Download: rabbitmq_email has not yet been built for unknown.
  • Authors: Petr Gotthard
  • GitHub: gotthardp/rabbitmq-email
rabbitmq_jsonrpc_channel
Adds an HTTP server which makes use of JSON-RPC polling to emulate AMQP channels in a web browser. Note that this functionality is split across various plugins:
  • Download: rfc4627_jsonrpc has not yet been built for unknown.
  • Download: rabbitmq_jsonrpc has not yet been built for unknown.
  • Download: rabbitmq_jsonrpc_channel has not yet been built for unknown.
  • Download: rabbitmq_jsonrpc_channel_examples has not yet been built for unknown. (to see example code)
  • Authors: The RabbitMQ Team
pgsql_listen_exchange
Translates PostgreSQL NOTIFY messages to AMQP messages and publishes them to bound queues. The PostgreSQL NOTIFY message channel is used as the routing key for the message using direct exchange style routing mechanics.

Note that two plugin files are required:

  • Download: epgsql has not yet been built for unknown.
  • Download: pgsql_listen_exchange has not yet been built for unknown.
  • Author: Gavin M. Roy
  • Github: aweber/pgsql-listen-exchange
